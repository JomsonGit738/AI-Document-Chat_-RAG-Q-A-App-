\# Project instructions



This is an Angular project.



Use Angular-specific skills and conventions when creating and editing this app.



\## Stack preferences



\- Use the latest stable Angular CLI.

\- Use TypeScript.

\- Use standalone components.

\- Prefer Angular signals where appropriate.

\- Prefer Angular control flow syntax such as `@if`, `@for`, and `@switch`.

\- Use Angular Router for routing.

\- Use reactive forms for non-trivial forms.

\- Use Angular testing patterns for unit tests.

\- Do not use React, JSX, Next.js, Vue, or non-Angular frontend patterns unless explicitly requested.



\## Package manager



Use npm unless the repo already contains another lockfile.



\## Quality checks



After creating or modifying code, run the relevant Angular commands when possible:

npm run build

npm test
